# Apple Design Resources Reference

Official source: https://developer.apple.com/design/resources/

Use this file when a review touches design assets, templates, icons, screenshots, or marketing surfaces.

## Resource Areas

- **iOS and iPadOS UI kits**: Use as references for current component scale, spacing, platform templates, and app icon construction.
- **App icon templates**: Check that icon artwork respects current platform template expectations and appearance-mode needs.
- **Fonts**: SF Pro is the system font for iOS, iPadOS, macOS, and tvOS. Use system text styles in app UI unless there is a strong brand/product reason not to.
- **SF Symbols**: Symbols should align with San Francisco text, support the needed weights/scales, adapt to reading direction when relevant, and carry accessible labels.
- **Icon Composer**: Use when reviewing layered icons, Liquid Glass-style icon output, appearance-mode annotations, or flattened export needs.
- **Product bezels, badges, and logos**: Use official resources when reviewing App Store screenshots, marketing images, device frames, Apple badges, or Apple technology logos.

## Review Checks

- Do not treat templates as screens to clone. Use them to verify scale, spacing, icon geometry, and asset construction.
- Check whether screenshots and marketing images accurately represent the app's actual UI and current capabilities.
- Verify that Apple badges, logos, device frames, and technology marks follow the relevant Apple-provided resource and identity guidance.
- Prefer SF Symbols over custom glyphs when the symbol fits the meaning and accessibility needs; require custom icons to match platform stroke, alignment, and scale.
